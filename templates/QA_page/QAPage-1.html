  • 4 Types of Search Results

    Keyword Input ID Input
    Term Query Documents with the most keywords in their titles and abstracts in common with the keywords in the entered query string.  The IQS concatenates all words in the title and abstract of the input document to generate a big query string. Documents with the most keywords in their titles and abstracts in common with that of the input document will be your search results. 
    Concept Query This mode is currently not supported because parsing the entered query string through Metamaplite takes too much time. The matching process is the same as “Term mode, keyword input”, except the entered query string is parsed through Metamaplite, and generated medical concepts are used as a transitional query string. The matching process is the same as “Term mode, ID input”, except the title and abstract of the input document are parsed through Metamaplite, and generated medical concepts are used as the new query string. This mode is supported because all documents have been pre-parsed through Metamaplite, saving response time after you click the search button.
